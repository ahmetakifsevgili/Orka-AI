# Codex Feature Workflow

Before any feature work:

1. Read `docs/project-state/current-roadmap.md`.
2. Read `docs/codex-skills/README.md`.
3. Read the relevant constitution file:
   - `backend-feature-constitution.md`
   - `ai-rag-feature-constitution.md`
   - `frontend-contract-constitution.md`
   - `data-lifecycle-constitution.md`
   - `testing-gate-constitution.md`
4. Produce a short plan before code changes.
5. Do not stage/commit unless explicitly requested.
6. Do not reorder roadmap without user approval.
7. During scoped feature work, do not introduce:
   - production enterprise hardening
   - frontend redesign
   - major AI/model integrations
   - large refactors
8. Stage 6B - Merkezi Sinavlar / Exam & Practice Content Engine is closed.
9. Current phase is post-6B productization / frontend-content readiness.
10. Do not start Stage 6C, global exam implementation, or teacher/institutional features without explicit user approval.
